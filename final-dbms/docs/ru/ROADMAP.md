# Roadmap

Этот документ — расширенный TODO‑лист. Он полезен, если вы хотите:

- продолжить развитие проекта,
- выбрать отдельную тему для изучения (оптимизация, индексы, транзакции),
- использовать репозиторий как ориентир для собственного учебного проекта.

## Индексы и оптимизация

- [ ] Персистентность BTREE: формат на диске, восстановление без пересборки, тесты на рестарт.
- [ ] Расширить выбор планов: больше правил для предикатов и составных условий.
- [ ] Cost‑based оптимизатор: статистики, оценка селективности, выбор операторов.
- [ ] Индексы по нескольким колонкам (composite).

## SQL‑диалект

- [ ] `UPDATE`
- [ ] `DELETE`
- [ ] `DROP TABLE`
- [ ] `DROP INDEX`
- [ ] `ORDER BY`
- [ ] `LIMIT`
- [ ] `JOIN` (начать с nested loop join)

## Транзакции и конкурентность

- [ ] Убрать глобальную сериализацию запросов (server‑level lock).
- [ ] Блокировки на уровне таблиц/страниц/строк.
- [ ] MVCC или упрощённая модель изоляции (например, Read Committed).
- [ ] Транзакционный лог (WAL) для durability.

## Storage engine

- [ ] Более строгие гарантии flush/close, тесты на crash‑recovery (в учебном виде).
- [ ] Улучшение управления свободным местом и компактизация страниц.
- [ ] Статистика по таблицам: количество строк, распределение значений.

## Инструменты и DX

- [ ] Сценарии нагрузочного тестирования (benchmark).
- [ ] Улучшение сообщений об ошибках (позиции в SQL, контекст).
- [ ] CI pipeline (сборка + тесты).
