# Contributing

Проект учебный, но pull requests и обсуждения приветствуются.

## Как предложить изменение

1. Создайте issue с описанием проблемы/идеи (что меняем и зачем).
2. Согласуйте подход (особенно если меняются протокол/формат данных/SQL‑диалект).
3. Подготовьте MR/PR с минимальным, атомарным изменением.

## Требования к изменениям

- Код должен собираться на Java 17.
- Должны проходить тесты:

```bash
./gradlew test
```

- Для функциональных изменений желательно добавить unit‑тест или e2e‑тест.
- Документацию в `README.md`/`docs/` нужно обновлять вместе с кодом, если меняется поведение.

## Стиль

- Предпочтение: маленькие классы с ясной ответственностью.
- Ошибки должны быть информативными (особенно на границах стадий pipeline).
- Не усложнять конкурентную модель без явной необходимости (см. `docs/ROADMAP.md`).
